\section{高频地波雷达工作原理及特点总结}

\subsection{工作原理}

高频地波雷达通常工作在 3--30 MHz 频段，属于高频超视距雷达的一类。该频段电磁波能够沿导电海面发生绕射传播，采用垂直极化天线时地波衰减相对较小，因此可以突破普通微波雷达的视距限制，对远距离海域进行连续观测。雷达发射高频电磁波后，海面、船舶和环境散射体会产生回波。接收端经过混频、采样、距离向处理、速度向处理和方位向处理后，可得到距离、径向速度、方位角、幅度和信噪比等点迹信息。

距离观测由电磁波往返传播时间决定。设电磁波传播速度为 \(c\)，目标距离为 \(R\)，回波时延为 \(\tau\)，则有
\begin{equation}
  \tau = \frac{2R}{c},
  \qquad
  R = \frac{c\tau}{2}.
  \label{eq:range}
\end{equation}

速度观测依赖多普勒频移。设雷达波长为 \(\lambda\)，径向速度为 \(v_r\)，多普勒频移为 \(f_d\)，单站雷达中二者满足
\begin{equation}
  f_d = \frac{2v_r}{\lambda},
  \qquad
  v_r = \frac{\lambda f_d}{2}.
  \label{eq:doppler}
\end{equation}

因此，数据中的速度字段本质上反映目标沿雷达到目标连线方向的速度分量，而不是完整二维速度。

从目标探测角度看，海上目标能够被高频地波雷达稳定发现通常需要满足两个基本条件：一是目标具有足够大的雷达散射截面，能够在海杂波和环境干扰背景中形成可检测回波；二是目标相对于雷达具有一定径向运动速度，使目标回波在多普勒域中与零多普勒附近的强背景分量分离。若目标回波弱或径向速度过小，即使空间位置上有点迹，也可能难以从海杂波背景中可靠确认。

方位估计主要依靠阵列接收信号之间的相位差和数字波束形成。若距离为 \(r\)，方位角为 \(\theta\)，则在局部平面坐标中可近似表示为
\begin{equation}
  x = r\sin\theta,\qquad y = r\cos\theta.
  \label{eq:xy}
\end{equation}

本次作业所用数据已经给出 \(x\)、\(y\)、\(v_x\)、\(v_y\) 等字段，因此后续算法直接在点迹层面开展目标候选提取和多帧关联。

\subsection{信号处理流程}

从原始回波到点迹数据，一般需要经历距离、速度、杂波抑制、检测、方位估计和跟踪等多个环节。工程处理中，地波雷达通常先对多通道阵列回波进行距离和多普勒处理，形成距离--多普勒谱，再进行海杂波、射频干扰和电离层杂波抑制，随后通过 CFAR 检测和参数估计输出目标点迹。本文可将高频地波雷达目标探测流程概括为图 \ref{fig:radar_pipeline}。

需要说明的是，本次实验使用的 \texttt{HFRData.mat} 已经不是最底层的原始回波或距离--多普勒谱数据，而是雷达系统处理后输出的点迹级数据。数据中保留了每个点迹的位置、速度、幅度和信噪比等字段，但不能据此反推出完整的谱级杂波抑制或 CFAR 检测过程。因此，本文的重点不是重新实现海杂波抑制、射频干扰抑制、电离层杂波抑制和谱级 CFAR 检测，而是在已有点迹基础上继续进行目标候选筛选、空间速度聚类和多帧连续性确认。

\subsection{技术特点}

高频地波雷达的主要优点是探测距离远、覆盖范围广、可对海面目标和海流信息进行连续观测。与普通视距雷达相比，地波传播使其适合海上大范围监测；与单帧成像类传感器相比，雷达还能提供速度、多普勒和时间连续性信息。

但高频地波雷达也存在明显限制。首先，海浪的一阶和二阶 Bragg 散射会形成强海杂波，一阶海杂波可能抬高检测基底，造成虚警或漏警；其次，高频雷达与其他射频信号共用 3--30 MHz 频带，容易受到射频干扰，同时电离层反射和散射也可能在距离和多普勒域产生展宽杂波；再次，目标回波可能较弱，在单帧中不一定形成稳定聚集。由于本次点迹数据缺少人工真值或 AIS 对照，难以直接计算严格意义上的准确率、召回率和 F1 值。因此，本文采用无监督质量评价，重点考察候选簇是否连续出现、航迹是否平滑、相邻帧跳变是否可控。

\section{地波雷达目标跟踪方法}

\subsection{地波雷达目标跟踪的基本原理}

目标检测可以抽象为二元假设检验。对某个观测向量 \(z\)，设
\begin{equation}
  H_0: z=n,\qquad H_1:z=s+n,
  \label{eq:hypothesis}
\end{equation}

其中 \(H_0\) 表示无目标，\(H_1\) 表示有目标，\(n\) 为噪声和杂波，\(s\) 为目标回波。若两类假设下的概率密度已知，可构造似然比
\begin{equation}
  \Lambda(z)=\frac{p(z|H_1)}{p(z|H_0)}.
  \label{eq:likelihood}
\end{equation}

当 \(\Lambda(z)>\eta\) 时判定为目标，其中 \(\eta\) 为检测门限。

实际雷达背景随距离、方位和海况变化，固定门限容易造成近距离强杂波误警或远距离弱目标漏检。因此，雷达处理中常使用恒虚警率检测思想。以 CA-CFAR 为例，若用 \(N\) 个参考单元估计背景功率，则
\begin{equation}
  \hat{P}_n = \frac{1}{N}\sum_{i=1}^{N}P_i,
  \qquad
  \gamma = \alpha\hat{P}_n.
  \label{eq:cfar}
\end{equation}

当待检测单元功率 \(P_0>\gamma\) 时，认为该单元存在显著目标。若参考单元服从指数分布，为满足虚警概率 \(P_{fa}\)，可取
\begin{equation}
  \alpha = N\left(P_{fa}^{-1/N}-1\right).
  \label{eq:cfar_alpha}
\end{equation}

在完整地波雷达处理中，CFAR 检测通常可分别在距离向和多普勒向进行，再综合两个维度的检测结果并通过峰值检测输出目标候选。常见方法包括 CA-CFAR、OS-CFAR 和自适应回归类 CFAR。由于本文数据已经是点迹，无法直接在原始距离--多普勒谱上重新进行谱级 CFAR。本文将 CFAR 的局部自适应思想迁移到点迹层面：每帧先按信噪比和幅度筛选较强点，再按距离分区修正局部门限，最后用空间、速度和信号强度特征进行密度聚类。

\subsubsection{自适应强点筛选}

单帧强点筛选是后续聚类和关联的基础。数据中的信噪比可理解为点迹相对于背景噪声或杂波的强弱估计，它是点迹质量指标，而不是目标真实性标签。若直接用固定 dB 阈值，容易受到帧间背景起伏和距离衰减影响；若简单放松全局阈值，又会显著增加杂波候选。因此，本文采用“全局分位阈值 + 距离分区局部修正 + 综合信号评分”的折中方案，在每帧内部保留相对更强的点迹，再交给聚类和多帧关联继续判别。

设第 \(k\) 帧中第 \(i\) 个点迹的信噪比和幅度分别为 \(\mathrm{snr}_{k,i}\) 和 \(A_{k,i}\)。首先按每帧计算全局分位阈值
\begin{equation}
  T_{\mathrm{snr}}^{(k)}=Q_{0.85}\left(\mathrm{snr}_{k}\right),
  \qquad
  T_{A}^{(k)}=Q_{0.65}\left(A_k\right).
  \label{eq:global_threshold}
\end{equation}

随后按距离将该帧点迹划分为 6 个分区，并在每个分区中计算局部阈值。局部阈值只用于适当降低弱背景区域准入门槛，不用于提高强背景区域门槛：
\begin{equation}
  T_{\mathrm{snr,local}}=\min\left(T_{\mathrm{snr}}^{(k)}, Q_{0.85}(\mathrm{snr}_{k,b})\right),
  \quad
  T_{A,\mathrm{local}}=\min\left(T_A^{(k)}, Q_{0.65}(A_{k,b})\right).
  \label{eq:local_threshold}
\end{equation}

同时定义综合信号评分
\begin{equation}
  S_{k,i}=R_{\mathrm{snr}}(k,i)+R_A(k,i),
  \label{eq:signal_score}
\end{equation}

其中 \(R_{\mathrm{snr}}\) 和 \(R_A\) 分别为该点在当前帧内的信噪比百分位和幅度百分位。若点迹满足局部信噪比和幅度同时过线，或 \(S_{k,i}\ge 1.75\)，则保留为强点。该设计用于补充保留单项特别强的点迹，同时避免大幅放松阈值导致候选簇爆炸。

\subsubsection{空间速度聚类}

对保留下来的强点构造特征向量
\begin{equation}
  q_i=\left[
  \frac{x_i}{L_p},
  \frac{y_i}{L_p},
  \frac{v_i}{L_v},
  \frac{\mathrm{snr}_i}{L_s},
  \frac{A_i}{L_A}
  \right]^T,
  \label{eq:feature}
\end{equation}

其中 \(L_p=10\ \mathrm{km}\)、\(L_v=15\ \mathrm{km/h}\)、\(L_s=6\ \mathrm{dB}\)、\(L_A=15\) 为归一化尺度。随后使用 DBSCAN 对强点进行聚类。对任一点 \(q_i\)，其 \(\varepsilon\) 邻域定义为
\begin{equation}
  N_{\varepsilon}(q_i)=\{q_j\mid \|q_i-q_j\|\le \varepsilon\}.
  \label{eq:dbscan}
\end{equation}

当 \(|N_{\varepsilon}(q_i)|\ge \mathrm{min\_samples}\) 时，\(q_i\) 为核心点。本文采用 \(\varepsilon=1.10\)、\(\mathrm{min\_samples}=4\)，并要求候选簇点数不少于 5。对每个候选簇，本文不再简单采用簇内点迹坐标的算术平均，而是根据信噪比和幅度构造正权重，并对位置、经纬度和速度进行加权平均：
\begin{equation}
  w_i =
  \left(\mathrm{snr}_i-\min(\mathrm{snr})+\epsilon\right)
  +
  \left(A_i-\min(A)+\epsilon\right),
  \qquad
  \bar{\mathbf{p}}=\frac{\sum_i w_i\mathbf{p}_i}{\sum_i w_i}.
  \label{eq:weighted_center}
\end{equation}

这样可以使簇中心更多受强回波点影响，减少弱散点对候选中心的扰动。

\subsubsection{多帧关联与航迹确认}

雷达目标跟踪通常包含航迹起始、航迹维持和航迹终止三个阶段，其中航迹维持需要不断进行状态预测、数据关联和状态更新。受参考程序中常速模型和 Kalman 滤波思想启发，本文没有直接对全部点迹开航迹，而是在 DBSCAN 候选簇层面引入简化常速 Kalman 预测门：未被关联的候选簇可形成新航迹头，已有航迹先用状态模型预测下一位置，再在波门内寻找满足速度和方向约束的候选簇。考虑到单帧候选可能短暂缺失，本文允许最多断开 1 帧后继续关联。

状态向量取为
\begin{equation}
  \mathbf{x}_t=[x_t,\ v_{x,t},\ y_t,\ v_{y,t}]^T,
  \label{eq:kalman_state}
\end{equation}

观测向量为候选簇中心
\begin{equation}
  \mathbf{z}_t=[x_t,\ y_t]^T.
  \label{eq:kalman_measurement}
\end{equation}

设相隔帧数为 \(\Delta k\)，相邻帧时间间隔为 \(\Delta t=60\ \mathrm{s}\)，速度单位为 \(\mathrm{km/h}\)，则常速预测可写为
\begin{equation}
  \hat{x}_t=x_{t-1}+v_{x,t-1}\frac{\Delta k\Delta t}{3600},
  \qquad
  \hat{y}_t=y_{t-1}+v_{y,t-1}\frac{\Delta k\Delta t}{3600}.
  \label{eq:predicted_position}
\end{equation}

Kalman 预测同时传播状态协方差，记预测观测为 \(\hat{\mathbf{z}}_t=H\hat{\mathbf{x}}_t\)，新息协方差为 \(S_t=H\hat{P}_tH^T+R\)，则候选簇与预测航迹之间的马氏距离为
\begin{equation}
  d_M^2=(\mathbf{z}_t-\hat{\mathbf{z}}_t)^T
  S_t^{-1}
  (\mathbf{z}_t-\hat{\mathbf{z}}_t).
  \label{eq:mahalanobis_gate}
\end{equation}

本文采用马氏距离阈值 \(d_M^2\le 9.21\) 作为主波门，同时保留 \(5.0\ \mathrm{km}\) 的欧氏距离上限以避免协方差过大导致波门过宽；若径向速度差不超过 \(15.0\ \mathrm{km/h}\)，\(v_x,v_y\) 合成速度差不超过 \(8.0\ \mathrm{km/h}\)，则认为当前候选落入跟踪门，并进入方向一致性判别。关联成功后，使用候选中心更新 Kalman 状态；若同一波门内存在多个候选，则选择马氏距离最小者。

连续两段中心位移向量分别为 \(\Delta\mathbf{p}_{t-1}\) 和 \(\Delta\mathbf{p}_t\)，转向角定义为
\begin{equation}
  \phi_t =
  \arccos
  \left(
  \frac{\Delta\mathbf{p}_{t-1}^{T}\Delta\mathbf{p}_{t}}
  {\|\Delta\mathbf{p}_{t-1}\|\|\Delta\mathbf{p}_{t}\|}
  \right).
  \label{eq:turn_angle}
\end{equation}

当相邻位移小于 0.25 km 时，不使用转向角判据，以避免短位移抖动放大角度误差；否则要求 \(\phi_t\le 125^\circ\)。

最近邻关联计算简单，适合课程作业中的点迹级后处理，但其局限也很明显：当同一跟踪门内存在多个背景候选或多个目标相互接近时，距离最近的候选不一定是真实目标回波，可能导致误关联或丢跟。因此，本文在最近邻关联之后继续使用长度、直线性、步长和转向角进行质量确认。

航迹确认采用长度和形态质量共同约束。候选航迹至少持续 4 帧，且满足
\begin{equation}
  \mathrm{straightness}
  =
  \frac{\|\mathbf{p}_{\mathrm{end}}-\mathbf{p}_{\mathrm{start}}\|}
  {\sum_{t=2}^{T}\|\mathbf{p}_{t}-\mathbf{p}_{t-1}\|}
  \ge 0.60.
  \label{eq:straightness}
\end{equation}

同时要求相邻帧最大步长不超过 2.00 km，平均步长不超过 1.00 km。该指标越接近 1，说明航迹越接近单调运动；若候选链虽然持续时间较长但来回绕动或突跳明显，则不确认为目标航迹。

质量确认参数主要决定最终候选链是否足够平滑。偏严方案只保留 2 条航迹，说明其可能漏掉短而可信的候选链；偏松方案虽然长度达标数量增加到 6 条，但质量确认数仍为 3，说明当前最终方案已经较好地过滤了跳变明显的候选。因此，本文最终参数不是单纯追求更多航迹，而是在候选数量、连续性和形态稳定性之间取得折中。

\subsubsection{结果边界与方法局限}

本文结果应表述为“疑似目标航迹”或“目标候选”，不能表述为已验证船舶真实航迹。原因在于：第一，本数据没有 AIS 真值或人工标注；第二，当前算法基于点迹后处理，而不是原始距离--多普勒谱上的完整杂波抑制、CFAR 检测和参数估计流程；第三，多帧关联虽已加入候选簇级常速 Kalman 预测和马氏距离波门，但仍采用最近邻分配，尚未使用概率数据关联或多假设跟踪处理多候选竞争问题。

对于图中较宽、较长的灰色点迹带，本文没有直接将其判定为航迹。这是因为这些区域虽然点迹密集，但多数点不满足“高信号、紧凑聚类、多帧连续和平滑移动”的联合条件。后续若需要进一步挖掘弱目标，可考虑引入 track-before-detect、概率数据关联、AIS 对照验证或人工标注样本，以区分真实目标和背景杂波结构。

\section{学习总结与体会}

通过本次大作业，我对高频地波雷达目标探测流程有了更系统的认识。雷达目标检测并不是简单地在散点图上寻找肉眼可见的密集区域，而是需要综合信号强度、空间聚集性、速度一致性和时间连续性。尤其是在点迹背景呈现宽带状和斑块状分布时，直观密集区域并不一定对应稳定目标；只有能够在相邻帧中连续出现并满足物理约束的候选链，才更适合被保留为疑似航迹。

本实验也说明，参数选择需要在漏检和虚警之间折中。过严的信噪比和幅度筛选会忽略局部弱信号区域，过松的筛选又会造成候选簇数量膨胀并增加误关联。通过敏感性分析可以看到，距离分区补偿比简单放松全局阈值更稳健，能够适当补充候选而不破坏最终质量确认结果。

在方法边界方面，本文没有将结果包装成“准确率”或“真实目标确认”，而是采用无监督质量指标解释结果。当没有 AIS 或人工标注作为真值时，算法输出只能说明候选链在当前规则下满足连续性和形态质量，不能证明其一定是船舶目标。后续可加入概率数据关联、多假设跟踪、AIS 验证和原始谱级检测，以提升工程可用性。

其中，核心算法主要集中在 \texttt{02\_detect\_and\_track.py}。该文件依次实现四个关键模块：第一，强点筛选模块按帧计算信噪比和幅度分位阈值，再按距离分区修正局部门限；第二，候选聚类模块将空间、速度、信噪比和幅度统一到可比较尺度后进行 DBSCAN 聚类，并用信噪比和幅度加权计算候选簇中心；第三，航迹关联模块在候选簇层面执行常速 Kalman 预测、马氏距离波门和最近邻关联；第四，质量确认模块根据持续帧数、直线性、步长和转向角筛选最终疑似航迹。

为体现核心处理逻辑，下面列出部分关键源码片段。代码片段 \ref{code:strong_filter} 展示了单帧强点筛选中的局部阈值与综合评分逻辑。

\begin{lstlisting}[style=PythonStyle,caption={强点筛选核心代码},label={code:strong_filter}]
# 计算单帧信噪比和幅度分位阈值
frame_snr_threshold = valid_table["snr"].quantile(SNR_QUANTILE)
frame_amp_threshold = valid_table["amp"].quantile(AMP_QUANTILE)
# 初始化每个点迹的局部筛选门限
valid_table["local_snr_threshold"] = frame_snr_threshold
valid_table["local_amp_threshold"] = frame_amp_threshold
# 计算点迹的综合信号评分
valid_table["snr_rank"] = valid_table["snr"].rank(pct=True)
valid_table["amp_rank"] = valid_table["amp"].rank(pct=True)
valid_table["signal_score"] = valid_table["snr_rank"] + valid_table["amp_rank"]
# 标记同时满足局部信噪比和幅度条件的点迹
local_signal_mask = (
    (valid_table["snr"] >= valid_table["local_snr_threshold"])
    & (valid_table["amp"] >= valid_table["local_amp_threshold"])
)
# 保留局部门限或综合评分达标的强点
combined_signal_mask = valid_table["signal_score"] >= SIGNAL_SCORE_MIN
strong_table = valid_table[local_signal_mask | combined_signal_mask].copy()
\end{lstlisting}

# 计算候选簇到预测状态的统计距离

# 同时通过距离、速度和方向门限时更新最近邻
if (
    distance <= best_distsance
    and spatial_distance <= MAX_LINK_DISTANCE_KM
    and velocity_diff <= MAX_VELOCITY_DIFF_KMH
    and vector_velocity_diff <= MAX_VECTOR_VELOCITY_DIFF_KMH
    and is_direction_consistent(candidate_row, track_state)

    best_index = candidate_index
    best_distance = distance
